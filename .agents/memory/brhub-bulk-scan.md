- **총괄표제부(getBrRecapTitleInfo)는 쓰지 말 것**: 해운대 우동 실측 recap 89건·숙박 0건 vs 표제부(getBrTitleInfo) 3,099건·집합+숙박 21건. 전국 발견 스캔은 표제부 전수만 신뢰 가능.
  **Why:** recap은 총괄표제부가 등록된 단지만 나오고 주용도 요약이 숙박 표기를 삼킴(엘시티도 미노출).
- **건축HUB는 2026 행정개편 신 법정동코드를 모름**: 전남광주통합특별시(12*) 조회는 오류 없이 totalCount=0 → 조용히 전 지역 누락. 폐지된 구코드(광주 29*, 전남 46*)로 조회해야 데이터가 나옴.
  **How to apply:** bjdong_codes.json 재생성 시 신코드(12*) 동/리를 이름 매칭으로 폐지 29*/46* 코드로 치환(2026-07 기준 3,176개 전부 매핑됨). 다른 API도 코드 개편 시 "0건=병합 신코드" 의심.
- 법정동코드 원천: code.go.kr POST codeFullDown.do(codeseId=법정동코드, euc-kr zip). 폐지 여부 컬럼 포함 — 매핑용으로 폐지분도 받아둘 것.
- totalCount가 문자열로 올 수 있음(int 캐스팅). 읍/면 상위 단위는 0건 — 리 레벨 순회 필수. numOfRows=100 정상 존중(숙박업 API와 다름).
- 분류는 building_registry._find_categories 재사용(관광숙박시설 함정 회피) + '생활형숙박시설' 변형 표기 별도 보강 필요.
- 재분류 배치에서 label=None은 두 부류: reason에 '실패/재시도' 포함 → 마커 없이 재시도 유지, 그 외 판정불가만 영구 마커. 안 나누면 일시 API 오류가 영구 미분류로 고착.
